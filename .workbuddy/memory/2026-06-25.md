### 4. GameSettings.gd 编译错误修复（6/26 晚）
- 问题：第 121 行 `_lang is String` 在 GDScript 4 中编译报错，因为 `_lang` 声明为 int 类型，编译器知道它不可能是 String
- 修复：改用 Variant 变量 `raw_lang` 接收 `get_value()` 返回值，用 `typeof(raw_lang) == TYPE_STRING` 做类型判断，再转换为 int 赋给 `_lang`
- 涉及文件：`Autoload/GameSettings.gd` 第 118-124 行
- 教训：GDScript 4 的静态类型检查比 3 严格，`is` 关键字在编译期检查类型兼容性，不能对已知类型变量做不可能的类型判断
