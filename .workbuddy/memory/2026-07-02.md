## 联机大厅5大BUG修复（08:15）

### 1. 踢人ENet报错 "max channels: 0"
- **原因**: `kick_player()` 先 `disconnectPeer(peer_id)` 再发 `.rpc()`，断开的 peer 已无通道
- **修复**: 将 `_sync_player_names.rpc()` 和准备状态同步移到 `disconnectPeer()` 之前

### 2. 被踢玩家无提示
- **原因**: `_notify_kicked()` 直接 emit `host_disconnected`，客户端被静默踢回选择界面
- **修复**: 新增 `kicked_from_room` 信号；LobbyUI `_show_kicked_dialog()` 弹出半透明遮罩 + 红色面板 "你已被踢出房间" + 确认按钮
- 新增翻译键: `kicked_msg` / `confirm_btn`

### 3. 准备状态不同步（客户端看不到自己已准备）
- **原因**: `_sync_ready_states` 全量替换 `player_ready` 字典，Godot RPC 序列化 Dictionary 时 int 键可能出问题
- **修复**: 新增 `_sync_player_ready(peer_id, is_ready)` 按个人同步，不再全量替换字典
- `toggle_ready()` 房主直接调 `_sync_player_ready.rpc()`，客户端调 `_request_ready.rpc_id(1)`
- `reset_ready()` / `kick_player()` 内的全量同步也改为遍历逐人 `_sync_player_ready.rpc()`

### 4. 单挑换队后无法开始
- **原因**: `_update_start_button()` 先检查 `all_ready()`，未全部准备时直接 return，永远不显示队伍冲突警告。用户看到"等待准备"误以为是队伍问题
- **修复**: 队伍冲突检查提前到 `all_ready()` 之前，冲突时优先显示"两队不能相同！"
- 乱斗/僵尸的颜色冲突同理提前

### 5. 开游戏后BGM不停
- **原因**: `BGMManager.stop_bgm()` 从未被调用
- **修复**: LobbyUI `_start_game_scene()` 和 MainMenu `_on_bot_mode_pressed()` / `_on_zombie_mode_pressed()` 入口处调 `BGMManager.stop_bgm()`

## 修复准备状态客户端看不到自己（11:20）

### 问题：客户端点准备，自己看到自己未准备，但房主看到已准备
- **根因**: `_update_player_list()` 用 `multiplayer.get_peers()` 获取玩家列表

- **修复**: 改用 `NetworkManager.player_names.keys()` 构建完整玩家ID列表，排序后统一渲染
- 同时给房主条目加 ★ 标记

## 修复联机开游戏队员进不去（14:58）

### 问题：主机开游戏后客户端不加载，报 "Node not found: Lobby" 和 cache ID 错误
- **根因**: `NetworkManager.start_game()` 是 `@rpc("authority", "call_remote")` — 无 `call_local`
  - `.rpc()` 只发远程，不在主机本地执行
  - 客户端执行时 `if not is_host: return` 直接返回
  - 内部的 `_notify_game_start.rpc()` **从未被调用**，客户端永远收不到游戏开始通知
  - 客户端留在大厅，主机进入游戏后 Lobby 节点已释放，客户端后续RPC找不到 Lobby 节点 → 报错
- **修复**:
  1. `LobbyUI._on_start_game_pressed()`: 直接调 `NetworkManager._notify_game_start.rpc(room_mode)`，跳过中间的 `start_game()`
  2. 加 `await get_tree().process_frame` 确保 RPC 发出后再切换场景
  3. 删除 `NetworkManager.gd` 中无用的 `start_game()` RPC 函数
- **修改文件**: `scripts/LobbyUI.gd` (1032-1047), `Autoload/NetworkManager.gd` (537-543)
