## 23:05 修复僵尸/Bot不攻击 + 设置返回逻辑错误

### BUG1：僵尸/Bot不攻击
1. **根因**
   - 游戏场景没有 NavigationMesh（导航网格）
   - `NavigationAgent2D` 在无导航网格时 `is_target_reachable()` 可能返回 true（或 false 但不可靠）
   - `is_navigation_finished()` 在无路径时立即返回 true
   - 导致 `_move_with_nav()` / `_nav_to()` 中 `velocity = Vector2.ZERO`，僵尸/Bot 原地不动

2. **修复**
   - `ZombieAI.gd` `_move_with_nav()` 增加三层保底：
     1. `is_target_reachable()` 返回 false → 直线追踪
     2. `is_navigation_finished()` 返回 true 但未到攻击范围 → 直线追踪
     3. `get_next_path_position()` 返回位置和当前位置几乎相同 → 直线追踪
   - `BotAI.gd` `_nav_to()` 增加相同三层保底逻辑
   - 保底函数 `_fallback_move()` 直接朝玩家方向移动

### BUG2：设置返回主菜单而不是暂停菜单
1. **根因**
   - 原实现用 `signal closed` + `settings_instance.closed.connect(_on_settings_closed)`
   - 信号连接在 `get_tree().paused = true` 的暂停状态下可能不可靠
   - 信号可能被覆盖或延迟触发

2. **修复**
   - `Settings.gd`：用 `var on_close_callback: Callable = Callable()` 替代 `signal closed`
   - `_on_back_pressed()`：调用 `on_close_callback.call()` 代替 `closed.emit()`
   - `BotGame.gd` / `ZombieGame.gd` / `MainMenu.gd`：设置 `settings_instance.on_close_callback = func(): _on_settings_closed()` 代替信号连接
   - 回调方式更直接可靠，不依赖信号系统的处理顺序

3. **验证**
   - 全项目 lint 0 错误
   - 待用户测试确认
